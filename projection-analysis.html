    Section 2.5 Analysis of Population Projections

    When we introduced projection functions that define a sequence recursively, \(x_{n+1} = f(x_n)\text{,}\) we discussed the relationship between fixed points of the projection function and equilibria of the sequence being studied. In this section, we will consider the analysis of projection functions in order to study the stability of equilibria. A stable equilibrium is likely to be observed in a physical setting. An unstable equilibrium is unlikely to be observed as small deviations from equilibrium push the system away from the steady state.
    We will first consider a prototype example, where the projection function is a linear function. That system generalizes the geometric sequence related to constant per capita growth. For more complex models, we will then discuss using a linear approximation near the fixed point (i.e., a tangent line) to analyze equilibrium stability. We conclude with a graphical representation of sequences generated by projection functions that is commonly referred to as a cobweb diagram.

    Subsection 2.5.1 Linear Projection Functions

    We begin by considering the consequence of using a linear function as a projection function. Mathematics texts usually introduce linear functions using the slope–intercept form, \(f(x)=mx+b\) where \(m\) and \(b\) are model parameters (slope and intercept). However, we will more frequently use the point–slope form, \(f(x)=m(x-a)+b\text{,}\) which has slope \(m\) and passes through a point \((a,b)\text{,}\) or more precisely, \(f(a)=b\text{.}\) As an additional warning, note in advance that we will often use symbols other than \(m\) to represent slope.
    We start by looking at some examples that have linear projection functions.

    Example 2.5.1.

    Consider a population model that uses a constant per capita growth rate \(r\) for which we introduce an emigration term that is constant, say \(E_{n} = m\) (for migration). Then our population model would be
    \begin{equation*} \hbox{New Population} = \hbox{Old Population} + \hbox{Natural Growth} - \hbox{Emigration}\text{.} \end{equation*}
    Using sequences, this would be written
    \begin{equation*} P_{n+1} = P_{n} + r \cdot P_n - m = (1+r) P_n - m\text{.} \end{equation*}
    The projection function for this model is linear with a slope \(\alpha = 1+r\) given by
    \begin{equation*} f(x) = \alpha x - m = (1+r)x - m\text{.} \end{equation*}

    Example 2.5.2.

    Consider a population that is decreasing by 5% per year. Suppose an intervention is established that supplements the population by 100 new individuals per year.
    Express a recursive sequence model for this population and identify the projection function.
    Solution.
    The recursive model for the population is motivated by the descriptive equation
    \begin{equation*} \hbox{New Population} = \hbox{Old Population} - \hbox{Natural Losses} + \hbox{Supplements}\text{.} \end{equation*}
    If \(P_n\) is the population after \(n\) years, then our recursive model is
    \begin{align*} P_{n+1} \amp = P_n - 0.05 P_n + 100\\ \amp = 0.95 P_n + 100 \end{align*}
    We identify the projection function \(P_{n+1} = f(P_n)\) as
    \begin{equation*} f(x) = 0.95 x + 100\text{,} \end{equation*}
    which is linear with slope \(\alpha=0.95\text{.}\)

    Example 2.5.3.

    A loan balance grows by \(1/12\) of the annual percentage rate each month. But a monthly payment reduces the loan balance. Write down a recursive model for the loan balance \(B_n\) where \(n\) is the number of months since the loan began, \(r\) is the annual percentage rate and \(p\) is the amount of the monthly payment. Then identify the projection function for this sequence.
    Solution.
    The recursive model for the loan balance is motivated by the descriptive equation
    \begin{equation*} \hbox{New Balance} = \hbox{Old Balance} + \hbox{Added Interest} - \hbox{Monthly Payment}\text{.} \end{equation*}
    Our recursive model is
    \begin{align*} B_{n+1} \amp = B_n + \frac{r}{12} B_n - m \\ \amp = (1+\frac{r}{12}) B_n - m \end{align*}
    The projection function is therefore
    \begin{equation*} f(x) = (1+\frac{r}{12}) x - m\text{,} \end{equation*}
    which is linear with slope \(\alpha = 1+\frac{r}{12}\text{.}\)
    The first step in analysis is to identify the fixed points which we use to rewrite our function. A linear projection function \(f(x)=\alpha x + \beta\) for constants \(\alpha\) (alpha) and \(\beta\) (beta) has exactly one fixed point so long as \(\alpha \ne 1\text{.}\) Recall that a fixed point is a solution \(x^*\) to the equation \(x=f(x)\text{:}\)
    \begin{gather*} x^* = \alpha x^* + \beta\\ x^*-\alpha x^* = \beta \\ (1-\alpha) x^* = \beta \\ x^* = \frac{\beta}{1-\alpha} = \frac{-\beta}{\alpha-1}. \end{gather*}
    Because it is a fixed point, we know \(f(x^*)=x^*\) and the slope is \(\alpha\text{.}\) Rewriting \(f(x)\) in point–slope form, we find the alternative representation
    \begin{equation*} f(x) = \alpha(x-x^*) + x^*\text{.} \end{equation*}
    The new representation allows us to rewrite our recursive equation for the sequence. Suppose that \(x_n\) refers to a sequence generated by the projection function \(x_{n+1} = f(x_n)\text{.}\) Using the new representation, this becomes
    \begin{equation*} x_{n+1} = \alpha (x_n-x^*) + x^*\text{.} \end{equation*}
    This is equivalent to the equation
    \begin{equation*} x_{n+1} - x^* = \alpha (x_n - x^*)\text{.} \end{equation*}
    If we define a new sequence \(z_n\) as the difference between \(x_n\) and the fixed point \(x^*\text{,}\)
    \begin{equation*} z_n = x_n - x^*\text{,} \end{equation*}
    then our equation is showing that the difference is a geometric sequence,
    \begin{equation*} z_{n+1} = \alpha z_n\text{.} \end{equation*}
    Geometric sequences have an explicit formula
    \begin{equation*} z_n = z_0 \cdot \alpha^n\text{,} \end{equation*}
    which leads to an explicit formula for \(x_n\) given by
    \begin{equation*} x_n = x^* + z_n = x^* + (x_0 - x^*) \cdot \alpha^n\text{.} \end{equation*}
    For this reason, I call such sequences a shifted geometric sequence, where the sequence has been shifted to be centered around the fixed point \(x^*\text{.}\)
    Now that we have an explicit formula for our sequence, we can analyze the stability. The geometric sequence \(z_n = z_0 \cdot \alpha^n\) corresponds to repeated multiplication by the same number \(\alpha\text{.}\) When \(\alpha \lt 0\) (negative), the sings of \(z_n\) alternate between positive and negative. When \(|\alpha| \gt 1\text{,}\) the sequence \(z_n\) grows in magnitude; when \(|\alpha| \lt 1\text{,}\) the magnitude of \(z_n\) converges to zero. Overall, the distance from the fixed point grows (\(|\alpha| \gt 1\)), or the distance decays (\(|\alpha| \lt 1\))), or the distance stays the same (\(|\alpha| = 1\)).

    Theorem 2.5.4. Stability of Fixed Points for Linear Projection Functions.

    For a projection function \(f(x)=\alpha x + \beta\) with fixed point
    \begin{equation*} \displaystyle x^* = \frac{\beta}{1-\alpha} = \frac{-\beta}{\alpha-1}\text{,} \end{equation*}
    the stability of the fixed point as an equilibrium for a sequence defined as \(x_{n+1} = f(x_n)\) is determined by the value of the slope \(\alpha\text{.}\)
    • If \(|\alpha| \lt 1\text{,}\) then every sequence converges \(x_n \to x^*\) and \(x^*\) is a locally stable equilibrium.
    • If \(|\alpha| \gt 1\text{,}\) then every sequence with \(x_0 \ne x^*\) diverges away from \(x^*\) and \(x^*\) is a locally unstable equilibrium.
    • If \(\alpha = -1\text{,}\) then the sequence \(x_n\) alternates between two values and \(x^*\) is neutrally stable as the sequence does not diverge.
    • If \(\alpha = 1\) and \(\beta=0\) (\(f(x)=x\)), then every number is a fixed point. If \(\beta \ne 0\text{,}\) then no fixed point exists.

    Subsection 2.5.2 Linearizing Projection Functions

    Most functions are not linear, so it might seem that the previous results are limited to just a few special cases. However, one of the basic premises of calculus is that many functions can be approximated by linear functions locally. That linear function corresponds to the tangent line, which exists so long as the function is differentiable.

    Theorem 2.5.7.

    If a function \(f\) is differentiable at \(x=c\) (that is, \(f'(c)\) exists), then we can define the linear approximation of \(f\) around \(x=c\text{.}\) The linearization is the linear function with slope \(f'(c)\) and passing through the point \((c,f(c))\) given by
    \begin{equation*} L_{f,c}(x) = f(c) + f'(c) \cdot (x-c)\text{.} \end{equation*}
    When \(x\) is sufficiently close to \(c\text{,}\) the functions are nearly indistinguishable.

    Proof.

    The tangent line equation is a consequence of standard calculus definitions. To clarify the idea of sufficiently close without trying to recreate an entire course in analysis, let us start by pointing out that because \(f'(c)\) exists, the average rate of change defined by \(\frac{f(x)-f(c)}{x-c}\) must converge to \(f'(c)\) as \(x\) approaches \(c\text{,}\) as given in the limit
    \begin{equation*} \lim_{x \to c} \frac{f(x)-f(c)}{x-c} = f'(c)\text{.} \end{equation*}
    This is equivalent to saying
    \begin{equation*} \lim_{x \to c} \left(\frac{f(x)-f(c)}{x-c} - f'(c)\right) = 0\text{.} \end{equation*}
    This can be rewritten as
    \begin{equation*} \lim_{x \to c} \frac{f(x)-L_{f,c}(x)}{x-c} = 0\text{.} \end{equation*}
    Because the denominator goes to zero, the numerator must go to zero even faster for the limit to equal zero.
    Now, suppose that we are working with a projection function \(f\) with a fixed point at \(x=x^*\text{.}\) This means that \(f(x^*) = x^*\text{.}\) The tangent line of the projection function around the fixed point will be
    \begin{equation*} L_{f,x^*}(x) = f(x^*) + f'(x^*) \cdot (x-x^*) = x^* + f'(x^*) \cdot (x-x^*)\text{.} \end{equation*}
    By construction, we know that \(x=x^*\) is a fixed point of the linearized projection function and the stability of the fixed point as an equilibrium is determined by the slope \(f'(x^*)\text{.}\) The following theorem guarantees that so long as \(|f'(x^*)| \ne 1\text{,}\) the stability of the fixed point for the projection function is the same as the linearized function. (There is no clear conclusion if \(|f'(x^*)| = 1\text{.}\))

    Theorem 2.5.8. Stability of Fixed Points for Projection Functions.

    Suppose a projection function \(f\) has a fixed point at \(x=x^*\) and \(f'(x^*)\) exists. So long as \(|f'(x^*)| \ne 1\text{,}\) the local stability of the fixed point is determined by the value of \(f'(x^*)\) as follows.
    • If \(|f'(x^*)| \lt 1\text{,}\) then \(x^*\) is a locally stable fixed point.
    • If \(|f'(x^*)| \gt 1\text{,}\) then \(x^*\) is a locally unstable fixed point.

    Subsection 2.5.3 Computer Assisted Analysis

    This course has the assumption that you have completed a first semester university calculus course. Therefore, it is assumed that you are able to calculate basic derivatives and integrals. However, there are many computational tools available that can compute these directly and help us avoid mistakes. Here, I introduce you to one such service called SageMath that supports an open-source mathematical programming language called Sage. You can get a blank Sage cell in which you can perform computations at the following link: https://sagecell.sagemath.org/. Note that from this site, you are actually able to choose from a variety of different languages. When doing algebraic manipulation of formulas as described here, be sure to choose the language Sage. This text embeds Sage cells within the document with the language set by default. (We have seen previous SageMath cells with the language set to R previously.)
    Sage is built on top of Python, and any valid Python code can be included with Sage commands. In our work with R, we have stored results in variables that represent memory locations to keep track of data. Sage allows us to create objects that represent constants, variables, and formulas in a mathematical sense. We then can manipulate those mathematical objects.
    Suppose we wanted to consider a projection function \(f(x) = x+0.2x(1-0.005x)\text{.}\) The tasks we want to accomplish are as follows:
    • Define our projection formula.
    • Solve for the fixed points of the formula.
    • Calculate the derivative of the projection formula.
    • Evaluate the derivative at each identified fixed point.
    After we see the mathematical results, we will apply Theorem 2.5.8 to determine stability.
    The process of defining a formula in Sage requires first declaring all math symbols that will be variables. In our example, the only variable is \(x\text{.}\) Sometimes we might have parameters that we leave in symbolic form and they would also need to be declared. The Sage command for declaring a variable is the function var(...), where the input consists of variable names in quotes and separated by commas. This command must be evaluated before we start using the symbols in a formula. For our problem, we will use the command var("x").
    To actually define our expression, we need to choose a name for the expression for later reference as a programming variable name. Where R prefers that we use the assignment operators <- or ->, Python (and therefore Sage) requires the assignment operator is always =, with the variable name on the left side of the statement. To declare our example as my_projection we would type the statement my_projection = x + 0.2*x*(1-0.05*x). Notice how we need to explicitly type * for multiplication.
    Sage is able to solve basic algebraic equations directly using the command solve(...). In the current problem, there is only one equation, but because we often want to solve a system of equations, we will put a list of equations with only one list inside square brackets. Because = is used for variable assignment, most computer languages including Python and R use a double equal == for creating equations that are true or false. A solution will be a specific value of the variable \(x\text{,}\) so we also need to specify what we are solving for. In the future, we will again be solving for multiple variables, we will again use a list of variables we are solving to find. Because we are solving \(f(x) = x\text{,}\) the basic command will therefore be solve([my_projection == x], [x]), and it returns a list of equations that solve the problem.
    Just as solve was the command to solve an equation, we will use a command derivative to find the derivative of our formula. Because there are potentially many symbols representing variables, we need to specify the symbol for the independent variable as part of the command. To differentiate \(f(x)\) with respect to \(x\text{,}\) we tell our expression object my_projection to perform an action derivative with respect to \(x\) using the syntax my_projection.derivative(x). If the variable was something other than \(x\text{,}\) we just change that part of the formula.
    We will store the results of solving the equation and finding the derivative so that we can use the results later. The last thing we need to do is to evaluate the derivative at the point of the solution to the fixed point equation. Suppose we named the result of our derivative f_prime. We need to tell that object to make a substitution using the subs command. We can also ask that the result is simplified as much as possible. Since we want to do this for each fixed point solution, we will do this step in a loop.
    The entire structure of our analysis is accomplished with the following commands.
    The previous code will show formulas giving
    \begin{equation*} f(x) = 0.2x(-0.005x + 1) + x \end{equation*}
    and
    \begin{equation*} f'(x) = -0.002 x + 1.200\text{.} \end{equation*}
    It identifies fixed points at \(x=0\) and \(x=200\text{.}\) Surprisingly, when evaluating \(f'(200)\text{,}\) it gives a value \(0.79999999999\text{.}\) This is a consequence of computers being unable to represent numbers in decimal form exactly. If you changed the code to use 2/10 instead of 0.2 and 5/1000 instead of 0.005, you will get exact results. That is, we find that \(f'(0) = \frac{6}{5} = 1.2\) and \(f'(200) = \frac{4}{5} = 0.8\text{.}\)
    It is our task to interpret the results. Because the slope of the projection function at the fixed point \(x=0\) is greater than 1, the fixed point \(x=0\) is a locally unstable fixed point. However, because \(f'(200)\) has a value 0.8 which is positive and less than 1, the fixed point \(x=200\) is locally stable. If a population starts small and close to \(x=0\text{,}\) the subsequent population values will move farther away from 0. (This is good for a population as it is increasing in size.) On the other hand, a population that is close to 200 will move closer to 200, as that fixed point is stable. The population value of 200 acts as a carrying capacity.

    Example 2.5.9.

    Use computer analysis to find the fixed points for the discrete logistic model \(P_{n+1} = (1+r_0)P_n - a P_n^2\text{.}\) Find the slopes \(f'(x^*)\) for each fixed point and then discuss how the stability depends on the values of the parameters.
    Solution.
    The projection function is \(f(x) = (1+r_0) x - a x^2\text{,}\) which is a function of \(x\) involving two additional parameters \(r_0\) and \(a\text{.}\) To perform our analysis using Sage, we will need to declare all three symbols as mathematical variables. Nevertheless, we can still solve the fixed point equation for \(x\text{,}\) and Sage will provide answers in terms of the paramters.
    When you evaluate that work, we will match results we would get using basic calculus. If \(f(x) = (1+r_0)x - ax^2\text{,}\) we know how to find a derivative as
    \begin{equation*} f'(x) = (1+r_0) - 2 a x\text{.} \end{equation*}
    Similarly, we can algebraically write down the fixed point equation and solve it by hand:
    \begin{align*} f(x) \amp = x \\ (1+r_0)x - a x^2 \amp = x \\ x+r_0 x - a x^2 \amp = x \\ r_0 x - a x^2 \amp = 0\\ x(r_0 - a x) \amp = 0 \end{align*}
    Because this equation says the product of \(x\) with \(r_0 - ax\) equals zero, the solutions are found by setting each factor individually to zero, giving us \(x=0\) and \(x = \frac{r_0}{a}\) as our two solutions. This matches what Sage tells us.
    From our Sage work, or by evaluating \(f'(0)\text{,}\) we find \(f'(0) = 1+r_0\text{.}\) This means that whenever \(r_0 \gt 0\text{,}\) \(f'(0) \gt 1\text{.}\) By Theorem 2.5.8, we now know that \(x^* = 0\) is an unstable fixed point whenever \(r_0 \gt 0\) It is actually possible to have \(-1 \lt r_0 \lt 0\) if the per capita death rate exceeds the per capita birth rate. In that circumstance, we have \(0 \lt f'(0) \lt 1\text{.}\) Consequently, \(x^*=0\) is a stable fixed when the death rate exceeds the birth rate for \(P=0\text{.}\)
    For the second fixed point \(x^* = \frac{r_0}{a}\text{,}\) we have \(f'(x^*) = 1 - r_0\text{.}\) We think through the same conditions as above. If \(-1 \lt r_0 \lt 0\) (negative \(r_0\)), we have \(f'(x^*) \gt 1\) but we also have a fixed point \(x^* \lt 0\) with a negative value. In the current setting for populations, \(P\lt 0\) does not make sense. But mathematically, such a negative fixed point would be unstable. If \(0 \lt r_0 \lt 1\text{,}\) we have \(f'(x^*)\) between 0 and 1, which means the fixed point is locally stable and sequences converge without oscillation. Once we have \(r_0 \gt 1\text{,}\) we have \(f'(x*) \lt 0\) so that nearby sequences oscillate above and then below the fixed point. For \(1 \lt r_0 \lt 2\text{,}\) we have \(|f'(x^*)| \lt 1\) and the oscillation decays away to the stable fixed point. Once we have \(r_0 \gt 2\text{,}\) this causes \(|f'(x^*)| \gt 1\) such that oscillation grows and the fixed point is locally unstable.
